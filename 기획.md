# MoneyTech - 재테크 유튜브 채널 분석 플랫폼

## 1. 프로젝트 개요

### 1.1 비전
재테크(주식, 코인, 부동산) 분야 유튜브 크리에이터와 콘텐츠를 데이터 기반으로 분석하여,
투자자들에게 **신뢰할 수 있는 정보 큐레이션**과 **재미있는 인사이트 대시보드**를 제공하는 서비스.

### 1.2 문제 정의
- 재테크 유튜브 채널이 폭발적으로 증가하고 있으나, **정보의 질을 판별하기 어렵다**
- 특정 종목/코인/지역을 추천하는 영상이 넘쳐나지만, **과거 적중률을 검증할 수 없다**
- 투자자들이 여러 채널을 돌아다니며 정보를 수집하는 데 **시간이 과도하게 소요된다**
- 기존 유튜브 분석 플랫폼(vling.net 등)은 마케팅 중심이며, **투자 콘텐츠 특화 분석이 없다**

### 1.3 핵심 차별점 (vs vling.net)
| 항목 | vling.net | MoneyTech |
|------|-----------|-----------|
| 타겟 | 광고주/MCN/유튜버 | 개인 투자자/재테크 관심자 |
| 분석 초점 | 채널 성과, 광고 단가 | 콘텐츠 신뢰도, 종목 적중률 |
| 카테고리 | 전 카테고리 | 재테크 특화 (주식/코인/부동산) |
| 콘텐츠 분석 | 메타데이터 위주 | 자막 기반 딥 분석 (종목 추출, 논조 분석) |
| 결과물 | 채널 리포트 | 투자 인사이트 대시보드, 트렌드 맵 |

## 2. 타겟 사용자

### 2.1 페르소나

**페르소나 A: 주식 초보 직장인 (30대)**
- 퇴근 후 유튜브로 재테크 공부
- "이 유튜버 말을 믿어도 되나?" 항상 고민
- 여러 채널의 의견을 빠르게 비교하고 싶음

**페르소나 B: 코인 투자자 (20~30대)**
- 빠르게 변하는 시장에서 실시간 트렌드 파악 필요
- 유튜버들이 언급하는 코인의 빈도와 타이밍에 관심
- 과거 추천 대비 실제 수익률 궁금

**페르소나 C: 부동산 관심자 (30~40대)**
- 특정 지역/아파트에 대한 유튜버들의 종합 의견 파악
- 장기적 시장 전망에 대한 다양한 관점 비교
- 전문가 채널 vs 일반 채널 구분 필요

## 3. 핵심 기능

### 3.1 Phase 1 - MVP (데이터 수집 + 기본 대시보드)

#### A. 유튜브 데이터 수집 파이프라인
```
[yt-dlp] → [메타데이터 추출] → [자막 다운로드] → [DB 저장]
```

**수집 대상 데이터:**
- 채널 정보: 채널명, 구독자 수, 총 조회수, 개설일, 업로드 빈도
- 영상 메타데이터: 제목, 설명, 태그, 조회수, 좋아요, 댓글 수, 업로드일, 길이
- 자막(subtitle): 자동 생성 자막 or 수동 자막 (한국어)
- 썸네일 이미지 URL

**수집 방법 (yt-dlp 활용):**
```bash
# 채널의 최근 영상 메타데이터 수집
yt-dlp --flat-playlist --dump-json "https://www.youtube.com/@channel_name"

# 개별 영상 자막 다운로드
yt-dlp --write-auto-sub --sub-lang ko --skip-download --write-info-json "VIDEO_URL"

# 메타데이터만 추출
yt-dlp --dump-json --no-download "VIDEO_URL"
```

**초기 수집 채널 목록 (분야별):**

| 분야 | 채널 예시 | 비고 |
|------|----------|------|
| 주식 | 삼프로TV, 신사임당, 슈카월드, 김작가TV | 대형 채널 중심 |
| 코인 | 코인데스크코리아, 비트코인갤러리, 크립토퀀트 | 뉴스+분석 혼합 |
| 부동산 | 부읽남, 빠숑, 월급쟁이부자들, 부동산읽어주는남자 | 전문가 채널 |
| 경제일반 | 머니투데이, 한경글로벌마켓, 이진우의 손에 잡히는 경제 | 매크로 관점 |

#### B. 자막 기반 콘텐츠 분석 (NLP)
- **종목/코인/지역 추출**: 자막에서 종목명, 코인명, 부동산 지역명 자동 추출
- **감성 분석**: 해당 종목/자산에 대한 긍정/부정/중립 논조 판별
- **키워드 트렌드**: 시기별 언급 빈도 변화 추적
- **요약 생성**: 영상 핵심 내용 3줄 요약 (LLM 활용)

#### C. 기본 대시보드
- **채널 랭킹**: 구독자, 조회수, 업로드 빈도별 재테크 채널 순위
- **트렌드 워드클라우드**: 최근 7일/30일 재테크 유튜브에서 가장 많이 언급된 키워드
- **종목 히트맵**: 유튜버들이 많이 언급하는 종목/코인을 히트맵으로 시각화
- **채널 비교**: 2~3개 채널의 주요 지표 나란히 비교

### 3.2 Phase 2 - 핵심 차별화 기능

#### D. 적중률 검증 시스템 (킬러 피처)
```
[유튜버 종목 추천 추출] → [추천 시점 주가/코인가 기록]
→ [1주/1개월/3개월 후 가격 비교] → [적중률 산출]
```
- 유튜버가 "이 종목 오를 겁니다" 라고 언급한 시점의 가격 기록
- 이후 실제 가격 변동과 비교하여 **채널별 적중률 스코어** 산출
- "이 유튜버의 최근 3개월 추천 적중률: 62%" 식으로 표시

#### E. 크로스 채널 의견 비교
- 동일 종목/코인/지역에 대해 여러 유튜버의 의견을 한 눈에 비교
- "삼성전자에 대한 의견": 채널A(매수), 채널B(관망), 채널C(매도) 시각화
- 의견 합의도(Consensus Score) 산출

#### F. 실시간 트렌드 알림
- 특정 종목이 갑자기 여러 채널에서 동시 언급되면 알림
- "지난 24시간 내 5개 이상 채널에서 언급된 종목" 피드
- 개인 관심 종목 등록 시 관련 신규 영상 알림

### 3.3 Phase 3 - 확장 기능

#### G. AI 투자 브리핑
- 매일 아침 "오늘의 재테크 유튜브 브리핑" 자동 생성
- 전날 업로드된 주요 영상의 핵심 내용 요약
- NotebookLM 연동으로 팟캐스트 스타일 오디오 브리핑 생성

#### H. 커뮤니티 기능
- 채널/영상에 대한 사용자 리뷰 및 평점
- "이 유튜버의 조언으로 수익을 봤다/손실을 봤다" 후기
- 유튜버 신뢰도 크라우드소싱

#### I. 포트폴리오 연동
- 사용자의 관심 종목 포트폴리오 등록
- 해당 종목을 언급하는 신규 영상 자동 큐레이션
- 포트폴리오 관련 유튜버 의견 종합 리포트

## 5. 기술 스택

### 5.1 데이터 수집 레이어
| 구성요소 | 기술 | 용도 |
|---------|------|------|
| YouTube 크롤러 | **yt-dlp** + Python | 영상 메타데이터, 자막 수집 |
| 스케줄러 | **APScheduler** / Cron | 주기적 데이터 수집 (1시간/6시간) |
| 주가 데이터 | **pykrx** / **yfinance** | 종목 가격 데이터 수집 |
| 코인 데이터 | **CoinGecko API** / **Upbit API** | 코인 가격 데이터 수집 |

### 5.2 데이터 처리/분석 레이어
| 구성요소 | 기술 | 용도 |
|---------|------|------|
| NLP 파이프라인 | **Claude API** / **OpenAI API** | 자막 분석, 종목 추출, 감성 분석, 요약 |
| 종목명 매칭 | **정규식** + **종목 사전** | 자막에서 종목/코인/지역명 인식 |
| 데이터 처리 | **Pandas** | 데이터 정제, 집계, 통계 |
| NotebookLM | **notebooklm-py** (비공식) | 영상 분석 외부 처리, 브리핑 생성 |

### 5.3 백엔드
| 구성요소 | 기술 | 용도 |
|---------|------|------|
| API 서버 | **FastAPI** (Python) | REST API 제공 |
| 데이터베이스 | **PostgreSQL** | 메인 데이터 저장 |
| 캐시 | **Redis** | 대시보드 데이터 캐싱 |
| 작업 큐 | **Celery** + Redis | 비동기 수집/분석 작업 |

### 5.4 프론트엔드
| 구성요소 | 기술 | 용도 |
|---------|------|------|
| 프레임워크 | **Next.js 15** (App Router) | SSR + SPA |
| UI 라이브러리 | **Tailwind CSS** + **shadcn/ui** | 컴포넌트 스타일링 |
| 차트 | **Recharts** / **D3.js** | 인터랙티브 데이터 시각화 |
| 상태 관리 | **Zustand** | 클라이언트 상태 |

### 5.5 인프라
| 구성요소 | 기술 | 용도 |
|---------|------|------|
| 컨테이너 | **Docker** + Docker Compose | 로컬 개발 및 배포 |

## 6. 데이터 모델 (주요 엔티티)

```
Channel (채널)
├── id, youtube_channel_id, name, subscriber_count
├── category (주식/코인/부동산/경제일반)
├── upload_frequency, total_views, created_at
├── trust_score, hit_rate
└── last_crawled_at

Video (영상)
├── id, channel_id, youtube_video_id, title
├── description, tags, view_count, like_count, comment_count
├── published_at, duration, thumbnail_url
├── subtitle_text (전체 자막)
├── summary (AI 생성 요약)
└── crawled_at

MentionedAsset (언급된 자산)
├── id, video_id, asset_type (stock/coin/real_estate)
├── asset_name, asset_code
├── sentiment (positive/negative/neutral)
├── context_text (언급 맥락 원문)
├── mentioned_at (영상 내 타임스탬프)
└── price_at_mention (언급 시점 가격)

Prediction (예측/추천 기록)
├── id, video_id, channel_id, asset_id
├── prediction_type (buy/sell/hold)
├── target_price, reason
├── predicted_at, evaluation_date
├── actual_price_after_1w, 1m, 3m
└── is_accurate (적중 여부)

DailyTrend (일별 트렌드)
├── id, date, category
├── top_keywords (JSON), top_assets (JSON)
├── total_videos_analyzed
└── sentiment_distribution (JSON)
```

## 7. 데이터 수집 파이프라인 상세

### 7.1 수집 흐름

### 7.2 yt-dlp 활용 전략

**Rate Limiting 대응:**
- 채널별 수집 간격 최소 5초
- IP 로테이션 또는 요청 분산 (필요시)
- YouTube API v3 병행 사용 (쿼터 내에서 메타데이터 보완)

**자막 품질 관리:**
- 자동 생성 자막의 경우 오탈자/오인식 보정 로직
- 자막 없는 영상: Whisper API로 음성→텍스트 변환 (Phase 2)

- **"이번 주 유튜브 재테크 요약"**: 가장 많이 언급된 종목 TOP 10

- **"숨은 보석 채널"**: 구독자는 적지만 적중률이 높은 채널 발굴

### 8.3 인터랙티브 요소

- **나만의 유튜버 포트폴리오**: 신뢰하는 유튜버 3~5명 선택 → 종합 의견 대시보드

```python
from notebooklm import NotebookLM

async def create_weekly_report(category: str, video_urls: list[str]):
    """주간 트렌드 리포트 자동 생성"""
    nlm = NotebookLM()

    # 1. 노트북 생성
    notebook = await nlm.create_notebook(
        title=f"MoneyTech 주간 리포트 - {category} - {date.today()}"
    )

    # 2. 유튜브 영상 소스 등록 (최대 20개)
    for url in video_urls[:20]:
        await nlm.add_source(notebook.id, url=url, source_type="youtube")

    # 3. 분석 요청
    analysis = await nlm.query(notebook.id,
        prompt=f"""
        이 영상들을 분석하여 다음을 정리해주세요:
        1. 이번 주 가장 많이 언급된 {category} 관련 종목/자산 TOP 5
        2. 각 종목/자산에 대한 유튜버들의 종합 의견 (매수/관망/매도)
        3. 이번 주 핵심 이슈 3가지
        4. 다음 주 주목해야 할 포인트
        """
    )

    # 4. 결과물 생성
    audio = await nlm.generate_audio(notebook.id)  # 팟캐스트
    infographic = await nlm.generate_artifact(
        notebook.id, artifact_type="infographic"
    )

    return {
        "analysis": analysis,
        "audio_url": audio.url,
        "infographic_url": infographic.url
    }

async def create_daily_briefing(videos: list[dict]):
    """일일 마켓 브리핑 자동 생성"""
    nlm = NotebookLM()

    notebook = await nlm.create_notebook(
```

## 12. yt-dlp 기능 최대 활용

### 12.1 핵심 활용 기능

| yt-dlp 기능 | 활용 방법 | 용도 |
|-------------|----------|------|
| `--flat-playlist` | 채널의 전체 영상 목록 빠르게 수집 | 신규 영상 감지 |
| `--dump-json` | 영상 메타데이터 JSON 추출 | 제목, 조회수, 좋아요, 댓글 수 등 |
| `--write-auto-sub` | 자동 생성 자막 다운로드 | 콘텐츠 분석의 핵심 데이터 |
| `--write-sub` | 수동 자막 다운로드 (있는 경우) | 더 정확한 텍스트 |
| `--sub-lang ko` | 한국어 자막 지정 | 한국 재테크 채널 타겟 |
| `--write-info-json` | 상세 메타데이터 JSON 파일 저장 | 채널 정보, 태그, 설명 등 |
| `--write-thumbnail` | 썸네일 이미지 다운로드 | 대시보드 표시용 |
| `--write-comments` | 댓글 데이터 수집 | 시청자 반응 분석 |
| `--date-range` | 날짜 범위 지정 수집 | 특정 기간 분석 |
| `--match-filter` | 조건부 필터링 | 조회수/길이 기준 필터 |
| `--playlist-items` | 최신 N개만 수집 | 효율적 증분 수집 |
| `--cookies-from-browser` | 브라우저 쿠키 활용 | 연령 제한 등 우회 |
| `--extract-audio` | 오디오만 추출 | Whisper STT 입력용 |

### 12.2 고급 수집 스크립트

```python
import subprocess
import json
from pathlib import Path

class YouTubeCrawler:
    """yt-dlp 기능을 최대한 활용하는 재테크 유튜브 크롤러"""

    def __init__(self, output_dir: str = "./data"):
        self.output_dir = Path(output_dir)

    def get_channel_videos(self, channel_url: str, limit: int = 50) -> list[dict]:
        """채널의 최근 영상 목록을 빠르게 수집"""
        cmd = [
            "yt-dlp",
            "--flat-playlist",
            "--dump-json",
            "--playlist-items", f"1:{limit}",
            "--no-warnings",
            channel_url
        ]
        result = subprocess.run(cmd, capture_output=True, text=True)
        return [json.loads(line) for line in result.stdout.strip().split('\n') if line]

    def get_video_detail(self, video_url: str) -> dict:
        """영상 상세 메타데이터 + 자막 + 썸네일 + 댓글 수집"""
        video_dir = self.output_dir / "videos"
        video_dir.mkdir(parents=True, exist_ok=True)

        cmd = [
            "yt-dlp",
            "--skip-download",           # 영상 다운로드 안 함
            "--write-auto-sub",          # 자동 자막
            "--write-sub",               # 수동 자막 (있으면)
            "--sub-lang", "ko",          # 한국어
            "--sub-format", "json3",     # 타임스탬프 포함 자막 포맷
            "--write-info-json",         # 상세 메타데이터
            "--write-thumbnail",         # 썸네일
            "--write-comments",          # 댓글
            "--extractor-args",
            "youtube:comment_sort=top;max_comments=100",  # 인기 댓글 100개
            "--output", str(video_dir / "%(id)s/%(id)s"),
            video_url
        ]
        subprocess.run(cmd, capture_output=True, text=True)

        # 결과 파싱
        return self._parse_video_data(video_dir, video_url)

    def search_videos(self, query: str, limit: int = 20) -> list[dict]:
        """유튜브 검색 결과 수집"""
        cmd = [
            "yt-dlp",
            "--flat-playlist",
            "--dump-json",
            "--playlist-items", f"1:{limit}",
            "--default-search", "ytsearch",
            f"ytsearch{limit}:{query}"
        ]
        result = subprocess.run(cmd, capture_output=True, text=True)
        return [json.loads(line) for line in result.stdout.strip().split('\n') if line]

    def get_subtitles_with_timestamps(self, video_url: str) -> list[dict]:
        """타임스탬프 포함 자막 추출 (종목 언급 시점 파악용)"""
        cmd = [
            "yt-dlp",
            "--skip-download",
            "--write-auto-sub",
            "--sub-lang", "ko",
            "--sub-format", "json3",     # 타임스탬프 포함
            "--output", "-",
            video_url
        ]
        # json3 포맷: 각 자막 세그먼트에 시작/종료 시간 포함
        # → 특정 종목이 언급된 정확한 타임스탬프 추출 가능
        pass

    def batch_collect(self, channel_urls: list[str], since_date: str = None):
        """다수 채널 일괄 수집 (증분 수집 지원)"""
        for url in channel_urls:
            cmd = [
                "yt-dlp",
                "--flat-playlist",
                "--dump-json",
                "--match-filter", f"upload_date >= {since_date}" if since_date else "",
                "--match-filter", "duration > 120",    # 2분 이상 영상만
                "--match-filter", "view_count > 1000", # 조회수 1000 이상만
                url
            ]
            # ... 수집 로직

    def extract_audio_for_stt(self, video_url: str, output_path: str):
        """자막 없는 영상: 오디오 추출 → Whisper STT"""
        cmd = [
            "yt-dlp",
            "--extract-audio",
            "--audio-format", "mp3",
            "--audio-quality", "5",     # 중간 품질 (STT용 충분)
            "--output", output_path,
            video_url
        ]
        subprocess.run(cmd, capture_output=True, text=True)
        # 이후 Whisper API로 텍스트 변환

    def get_channel_info(self, channel_url: str) -> dict:
        """채널 메타 정보 수집"""
        cmd = [
            "yt-dlp",
            "--dump-json",
            "--playlist-items", "0",
            channel_url
        ]
        result = subprocess.run(cmd, capture_output=True, text=True)
        data = json.loads(result.stdout)
        return {
            "channel_name": data.get("channel"),
            "channel_id": data.get("channel_id"),
            "subscriber_count": data.get("channel_follower_count"),
            "description": data.get("description"),
            "tags": data.get("tags", []),
        }

    def monitor_new_uploads(self, channel_url: str, last_video_id: str) -> list[dict]:
        """신규 업로드 모니터링 (증분 수집)"""
        cmd = [
            "yt-dlp",
            "--flat-playlist",
            "--dump-json",
            "--playlist-items", "1:10",  # 최신 10개만 체크
            "--break-on-existing",       # 이미 수집한 영상 만나면 중단
            channel_url
        ]
        result = subprocess.run(cmd, capture_output=True, text=True)
        videos = [json.loads(line) for line in result.stdout.strip().split('\n') if line]
        # last_video_id 이후 영상만 반환
        return [v for v in videos if v.get("id") != last_video_id]
```

### 12.3 yt-dlp 활용 특수 기능

**A. 댓글 분석을 통한 시청자 반응 파악**
```bash
# 인기 댓글 수집 → 시청자들의 실제 반응 분석
yt-dlp --write-comments --extractor-args \
  "youtube:comment_sort=top;max_comments=200" \
  --skip-download VIDEO_URL
```
- 시청자들이 유튜버 추천에 동의하는지/반대하는지 감성 분석
- "덕분에 수익 봤다" vs "이거 믿고 물렸다" 패턴 감지

**B. 자막 타임스탬프 기반 종목 언급 시점 추적**
```bash
# json3 포맷으로 정확한 타임스탬프 확보
yt-dlp --write-auto-sub --sub-format json3 --sub-lang ko \
  --skip-download VIDEO_URL
```
- 영상 어느 시점에서 특정 종목을 언급했는지 정확히 파악
- 대시보드에서 "해당 시점으로 바로 이동" 링크 제공

**C. 날짜/조건 기반 스마트 수집**
```bash
# 특정 기간 + 조건 필터링
yt-dlp --flat-playlist --dump-json \
  --dateafter 20260301 --datebefore 20260307 \
  --match-filter "view_count > 5000" \
  --match-filter "duration > 300" \
  CHANNEL_URL
```

**D. 라이브 스트림 아카이브 수집**
```bash
# 재테크 라이브 방송 (삼프로TV 등) 자막 수집
yt-dlp --write-auto-sub --sub-lang ko --skip-download \
  --match-filter "was_live = True" \
  CHANNEL_URL
```
