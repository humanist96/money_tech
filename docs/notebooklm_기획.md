# MoneyTech × NotebookLM 자동 리서치 기획 v3

## 테스트 결과 기반 전략 변경

v2에서는 "결과물 소비는 NotebookLM 웹으로 유도"였으나,
**2026-03-09 실제 테스트에서 모든 기능(Q&A, 오디오, 보고서, 퀴즈)이 API로 정상 동작** 확인.

→ **변경: 수집 + 분석 + 결과 확인까지 MoneyTech UI 안에서 수행**
→ NotebookLM 웹 링크는 "더 자세히 보기" 보조 수단으로만 제공

### 검증된 API 기능 (notebooklm-py 호환)

| 기능 | RPC | 테스트 결과 | UI 표시 |
|------|-----|------------|---------|
| 노트북 생성 | `CCqFvf` | ✅ 1.1s | - |
| YouTube URL 소스 추가 | `izAoDd` | ✅ (경고 나오나 실제 추가됨) | 진행률 |
| 텍스트 소스 추가 | `izAoDd` | ✅ 2.0s | 진행률 |
| 소스 목록 확인 | `rLM1Ne` | ✅ 0.4s | 소스 카드 |
| **Q&A 채팅** | `GenerateFreeFormStreamed` | ✅ 7.5s, 한국어, 출처 인용 | **채팅 UI** |
| **오디오 브리핑** | `R7cb6c` (type=1) | ✅ 비동기 생성 | 상태 + NB 링크 |

## 핵심 UX 원칙

### 1. Q&A는 인라인, 산출물은 링크

| 기능 | UI 방식 | 이유 |
|------|---------|------|
| **Q&A** | MoneyTech 내 채팅 UI | 7.5초 응답, 실시간 가능, 가장 자주 사용 |
| **오디오** | 생성 요청 + NB 링크 | 3~5분 소요, 재생은 NB 웹이 최적 |
| **보고서/가이드** | 생성 요청 + NB 링크 | 문서 렌더링은 NB 웹이 최적 |
| **퀴즈** | 생성 요청 + NB 링크 | 인터랙티브 UI는 NB 웹이 최적 |

### 2. 산출물 상태 모니터링

```
생성 요청 → [생성 중... ⏳] → [완료 ✅ - NB에서 확인 ↗]
              │                    │
              │ 폴링 (10초 간격)    │ artifacts.list()로 상태 확인
              └────────────────────┘
```

- 생성 요청 후 `artifacts.list()`를 10초 간격으로 폴링
- status가 `completed`(3)이면 "완료" 표시 + NotebookLM 링크 활성화
- 링크: `https://notebooklm.google.com/notebook/{id}`

### 3. NotebookLM 링크 전략

모든 산출물 옆에 NotebookLM 직접 링크 제공:
- 오디오: NB에서 바로 재생 가능
- 보고서: NB에서 Google Docs로 내보내기 가능
- 퀴즈: NB에서 인터랙티브하게 풀기 가능

```
NotebookLM 링크 형식:
  https://notebooklm.google.com/notebook/{notebook_id}
```

## 기술 아키텍처 (v3)

### 데이터 플로우

```
[사용자] "클로드가 왜 좋은지" + [리서치 시작]
    │
    ▼ (프론트엔드에서 병렬 수집 - 각각 별도 API 호출)
    │
    ├─→ [GET /api/search/youtube?q=클로드+AI]
    │     → YouTube 영상 10개 URL (기존 API 재활용)
    │
    ├─→ [GET /api/research/news?q=클로드+AI]  ★ 신규
    │     → 뉴스 기사 10개 URL (Naver/Google RSS)
    │
    ├─→ [GET /api/research/db-insights?q=클로드]  ★ 신규
    │     → DB 채널 분석 데이터 → 텍스트 생성
    │
    └─→ [POST /api/search/report]  (기존 확장)
          → GPT-4o-mini 종합 리포트 텍스트
    │
    ▼ (수집 완료 → Chrome 확장 프로그램으로 NotebookLM 주입)
    │
    ├─→ extensionCall('createNotebook', { title })
    ├─→ extensionCall('addSourceUrl', { url }) × N (YouTube)
    ├─→ extensionCall('addSourceUrl', { url }) × N (뉴스)
    ├─→ extensionCall('addSourceText', { content }) × 2 (DB분석 + AI리포트)
    │
    ▼ (주입 완료 → 15초 대기 후 소스 처리 확인)
    │
    └─→ 3단계 결과 화면 표시
         ├─→ Q&A 채팅 (extensionCall('chat'))
         ├─→ 오디오 생성 (extensionCall('generateAudio'))
         ├─→ 보고서 생성 (extensionCall('generateReport'))
         ├─→ 퀴즈 생성 (extensionCall('generateQuiz'))
         └─→ 아티팩트 상태 폴링 (extensionCall('getArtifactStatus'))
```

### Vercel 제약 대응

| 제약 | 해결책 |
|------|--------|
| 10초 serverless 타임아웃 | 각 수집을 개별 API로 분리, 프론트에서 병렬 호출 |
| Python 불가 | 뉴스 API를 TypeScript로 구현 |
| NotebookLM 인증 | Chrome 확장 프로그램 (사용자 쿠키 사용) |
| 오디오 생성 3~5분 | 비동기 요청 + 프론트에서 상태 폴링 |

## 핵심 원칙 요약

1. **Q&A는 MoneyTech UI 안에서** - 가장 빈번한 사용, 7.5초 응답
2. **산출물(오디오/보고서/퀴즈)은 생성 요청 + NotebookLM 링크** - 비동기 생성, NB 웹이 최적
3. **데이터 수집은 Vercel API** - YouTube/뉴스/DB 병렬 수집
4. **NotebookLM 주입은 Chrome 확장** - 사용자 쿠키 기반 인증
5. **모든 단계에서 NotebookLM 직접 링크 제공** - 파워유저용 탈출구
