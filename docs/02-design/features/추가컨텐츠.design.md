## 컨텐츠 1: 유튜버 적중률 리더보드

### API 설계

```
GET /api/leaderboard?period=1w|1m|3m&category=all|stock|coin|real_estate|economy
```

**SQL 쿼리:**
```sql
SELECT
  c.id, c.name, c.thumbnail_url, c.category,
  COUNT(p.id) as total_predictions,
  COUNT(p.id) FILTER (WHERE p.is_accurate = true) as accurate_count,
  ROUND(100.0 * COUNT(p.id) FILTER (WHERE p.is_accurate = true) / NULLIF(COUNT(p.id), 0)) as hit_rate,
  -- 연속 적중 (최근 예측에서 연속 true 카운트)
  -- 최근 10건 성적
  json_agg(
    json_build_object('type', p.prediction_type, 'accurate', p.is_accurate, 'asset', ma.asset_name)
    ORDER BY p.predicted_at DESC
  ) FILTER (WHERE p.predicted_at IS NOT NULL) as recent_predictions
FROM channels c
JOIN predictions p ON p.channel_id = c.id
LEFT JOIN mentioned_assets ma ON ma.id = p.mentioned_asset_id
WHERE p.is_accurate IS NOT NULL
  AND (p.actual_price_after_1w IS NOT NULL)  -- period 별 분기
GROUP BY c.id
HAVING COUNT(p.id) >= 3  -- 최소 3건 이상
ORDER BY hit_rate DESC, total_predictions DESC
LIMIT 30
```

## 컨텐츠 2: 종목 컨센서스 대시보드

### API 설계

```
GET /api/consensus?minMentions=3&category=all|stock|coin&sort=score|mentions|controversy
```

**SQL 쿼리:**
```sql
SELECT
  ma.asset_name, ma.asset_code, ma.asset_type,
  COUNT(DISTINCT v.channel_id) as channel_count,
  COUNT(*) as total_mentions,
  ROUND(100.0 * COUNT(*) FILTER (WHERE ma.sentiment = 'positive') / NULLIF(COUNT(*), 0)) as positive_pct,
  ROUND(100.0 * COUNT(*) FILTER (WHERE ma.sentiment = 'negative') / NULLIF(COUNT(*), 0)) as negative_pct,
  ROUND(100.0 * COUNT(*) FILTER (WHERE ma.sentiment = 'neutral') / NULLIF(COUNT(*), 0)) as neutral_pct,
  -- 컨센서스 점수: 의견 일치도 (0=완전분산, 100=완전일치)
  GREATEST(
    ROUND(100.0 * COUNT(*) FILTER (WHERE ma.sentiment = 'positive') / NULLIF(COUNT(*), 0)),
    ROUND(100.0 * COUNT(*) FILTER (WHERE ma.sentiment = 'negative') / NULLIF(COUNT(*), 0))
  ) as consensus_score,
  -- 최근 7일 감성 변화
  json_agg(DISTINCT v.channel_id) as mentioning_channels
FROM mentioned_assets ma
JOIN videos v ON v.id = ma.video_id
WHERE v.published_at > NOW() - INTERVAL '30 days'
GROUP BY ma.asset_name, ma.asset_code, ma.asset_type
HAVING COUNT(DISTINCT v.channel_id) >= 2
ORDER BY consensus_score DESC
```

## 컨텐츠 12: 신규 종목 탐지기

### API 설계

```
GET /api/emerging?days=7&minChannels=1
```

**SQL 쿼리:**
```sql
-- 최근 7일 내 처음 언급된 종목
WITH first_mentions AS (
  SELECT
    ma.asset_name, ma.asset_code, ma.asset_type,
    MIN(v.published_at) as first_mentioned_at,
    COUNT(DISTINCT v.channel_id) as channel_count,
    COUNT(*) as mention_count,
    MODE() WITHIN GROUP (ORDER BY ma.sentiment) as dominant_sentiment,
    json_agg(DISTINCT c.name) as channels
  FROM mentioned_assets ma
  JOIN videos v ON v.id = ma.video_id
  JOIN channels c ON c.id = v.channel_id
  GROUP BY ma.asset_name, ma.asset_code, ma.asset_type
  HAVING MIN(v.published_at) > NOW() - INTERVAL '7 days'
),
-- 언급 가속도 (일별 증가율)
daily_counts AS (
  SELECT
    ma.asset_name,
    DATE(v.published_at) as mention_date,
    COUNT(*) as daily_count
  FROM mentioned_assets ma
  JOIN videos v ON v.id = ma.video_id
  WHERE v.published_at > NOW() - INTERVAL '7 days'
  GROUP BY ma.asset_name, DATE(v.published_at)
)
SELECT fm.*, dc_trend
FROM first_mentions fm
LEFT JOIN LATERAL (
  SELECT json_agg(json_build_object('date', mention_date, 'count', daily_count) ORDER BY mention_date) as dc_trend
  FROM daily_counts WHERE asset_name = fm.asset_name
) t ON true
ORDER BY fm.channel_count DESC, fm.mention_count DESC
```

**응답 타입:**
```typescript

  assetName: string

  dominantSentiment: string
  channels: string[]
  dailyTrend: Array<{ date: string; count: number }>
```

```sql
-- 같은 채널이 같은 종목에 대해 감성이 바뀐 경우
WITH channel_asset_opinions AS (
  SELECT
    c.id as channel_id, c.name as channel_name,
    ma.asset_name, ma.asset_code,
    ma.sentiment,
    v.published_at,
    v.title as video_title,
    v.youtube_video_id,
    LAG(ma.sentiment) OVER (
      PARTITION BY c.id, ma.asset_name

FROM channel_asset_opinions
WHERE prev_sentiment IS NOT NULL
  AND prev_sentiment != sentiment
ORDER BY published_at DESC
```

## 컨텐츠 16: 댓글 감성 분석 (신규)

### 개요

유튜브 영상의 **상위 댓글**을 수집하여 시청자 반응을 분석.
유튜버 의견(영상 내용)과 시청자 의견(댓글)의 **괴리**를 탐지.

### 데이터 수집 (Crawler 확장)

**수집 방법:** yt-dlp (API 쿼터 무소모, 이미 설치됨)

**새 DB 테이블:**
```sql
-- 004_comments.sql
CREATE TABLE IF NOT EXISTS video_comments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  video_id UUID REFERENCES videos(id) ON DELETE CASCADE,
  youtube_comment_id TEXT UNIQUE,
  author_name TEXT,
  text TEXT NOT NULL,
  like_count INTEGER DEFAULT 0,
  reply_count INTEGER DEFAULT 0,
  sentiment TEXT CHECK (sentiment IN ('positive','negative','neutral')),
  mentioned_assets JSONB DEFAULT '[]',
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_comments_video ON video_comments(video_id);
CREATE INDEX idx_comments_sentiment ON video_comments(sentiment);
CREATE INDEX idx_comments_likes ON video_comments(like_count DESC);
```

**Crawler 추가 (youtube.py):**
```python
def get_video_comments(video_id, max_comments=50):
    """yt-dlp로 상위 댓글 추출 (좋아요 순)"""
    url = f"https://www.youtube.com/watch?v={video_id}"
    cmd = [
        "yt-dlp", "--skip-download",
        "--write-comments", "--extractor-args",
        f"youtube:max_comments=all,{max_comments},all,0",
        "--dump-json", url
    ]
    # ... parse comments from JSON output
    # 각 댓글에 대해 asset_dictionary.find_assets_in_text() 실행
    # sentiment 분석 수행
```

**수집 전략:**
- 최근 7일 영상의 상위 50개 댓글만 수집 (좋아요 순)
- 크롤링 주기: 일 1회 (기존 6시간 크론잡과 분리)
- 예상 소요: 74채널 × 평균 3영상/주 × 50댓글 = ~11,000 댓글/주

### API 설계

```
GET /api/comments/analysis?videoId=xxx
  → 영상별 댓글 감성 분포 + 상위 댓글

GET /api/comments/divergence?days=7
  → 유튜버 의견 vs 시청자 의견 괴리 종목
```

**괴리 탐지 SQL:**
```sql
-- 유튜버가 긍정인데 시청자(댓글)가 부정인 경우
SELECT
  ma.asset_name, ma.asset_code,
  -- 유튜버 감성 (영상 내 언급)
  ROUND(100.0 * COUNT(ma.id) FILTER (WHERE ma.sentiment = 'positive')
    / NULLIF(COUNT(ma.id), 0)) as youtuber_positive_pct,
  -- 시청자 감성 (댓글)
  ROUND(100.0 * COUNT(vc.id) FILTER (WHERE vc.sentiment = 'positive')
    / NULLIF(COUNT(vc.id), 0)) as viewer_positive_pct,
  -- 괴리도 (0=일치, 100=완전반대)
  ABS(
    ROUND(100.0 * COUNT(ma.id) FILTER (WHERE ma.sentiment = 'positive') / NULLIF(COUNT(ma.id), 0))
    - ROUND(100.0 * COUNT(vc.id) FILTER (WHERE vc.sentiment = 'positive') / NULLIF(COUNT(vc.id), 0))
  ) as divergence_score
FROM mentioned_assets ma
JOIN videos v ON v.id = ma.video_id
LEFT JOIN video_comments vc ON vc.video_id = v.id
  AND vc.mentioned_assets @> jsonb_build_array(jsonb_build_object('name', ma.asset_name))
WHERE v.published_at > NOW() - INTERVAL '7 days'
GROUP BY ma.asset_name, ma.asset_code
HAVING COUNT(DISTINCT vc.id) >= 5  -- 최소 댓글 5개 이상
ORDER BY divergence_score DESC
```

## 컨텐츠 17: 댓글 기반 투자 시그널 (대시보드 통합)

### 개요

댓글 데이터에서 추출한 시그널을 기존 대시보드에 위젯으로 추가.

### 시그널 종류

| 시그널 | 조건 | 의미 |
|--------|------|------|
| **댓글 폭증** | 댓글수 3일 평균 대비 3배+ | 시장 관심 급증 |
| **감성 급변** | 댓글 긍정률 전주 대비 20%p+ 변화 | 분위기 전환 |
| **유튜버-시청자 괴리** | 괴리도 40점+ | 시장 인식 불일치 |
| **댓글 내 종목 신규 등장** | 댓글에서 처음 언급된 종목 | 풀뿌리 관심 시작 |

### 대시보드 위젯

```
┌──────────────────────────────┐
```
