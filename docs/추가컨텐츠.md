## 0. 핵심 방향 전환

### 문제 인식

기존 기획은 74개 전체 채널을 대상으로 했으나, 대부분의 컨텐츠(적중률, 포트폴리오, 역발상 등)는
**실제로 매수/매도를 권유하고 가격을 예측하는 채널**에서만 의미가 있다.

- 슈카월드, 언더스탠딩, 머니투데이 → **경제 해설/뉴스** (예측 아닌 설명)
- 한국경제TV, SBS비즈 → **언론 미디어** (중립적 보도)
- 주식단테, 소수몽키, 찰리브라웅 → **리딩방/예측형** (매수/매도 직접 권유)

**→ "리딩방/예측형 유튜버"를 자동 분류하고, 이들에 한정한 컨텐츠를 기획한다.**

## 1. 채널 분류 체계 (Channel Classification)

### 1.1 채널 유형 정의

| 유형 | 코드 | 설명 | 예시 |
|------|------|------|------|
| **예측형 (Predictor)** | `predictor` | 특정 종목 매수/매도/목표가 직접 제시 | 주식단테, 소수몽키, 찰리브라웅 |
| **리딩방 (Leader)** | `leader` | 실시간 매매 시그널, 종목 리딩 | (발굴 대상) |
| **해설형 (Analyst)** | `analyst` | 시장 해설/교육, 직접 추천 없음 | 슈카월드, 언더스탠딩 |
| **미디어 (Media)** | `media` | 뉴스/보도, 객관적 전달 | 머니투데이, 한국경제TV |

> **핵심 대상**: `predictor` + `leader` 유형만 예측 관련 컨텐츠에 포함

### 1.2 자동 분류 스코어링 시스템

채널별로 최근 30일 영상을 분석하여 **예측 강도 점수(Prediction Intensity Score, PIS)** 산출:

```
PIS = (P1 × 0.30) + (P2 × 0.25) + (P3 × 0.20) + (P4 × 0.15) + (P5 × 0.10)
```

| 지표 | 코드 | 측정 방법 | 점수 범위 |
|------|------|----------|----------|
| **예측 밀도** | P1 | 예측 발생 영상 수 / 전체 영상 수 | 0~100 |
| **행동 키워드 강도** | P2 | 영상당 평균 BUY/SELL 패턴 매칭 수 | 0~100 (cap) |
| **종목 집중도** | P3 | 상위 5개 종목이 전체 언급의 몇 %인지 | 0~100 |
| **목표가 빈도** | P4 | "목표가", "XX원", "XX달러" 언급 영상 비율 | 0~100 |
| **감성 편향도** | P5 | |긍정% - 부정%| 평균 (중립이 아닌 정도) | 0~100 |

#### P1: 예측 밀도 (Prediction Density)

```python
# 기존 predictions 테이블 활용
prediction_density = (영상 중 prediction 1개+ 있는 영상 수) / (전체 영상 수) × 100
```

- 예측형 채널: 60~90% (대부분의 영상에서 예측)
- 해설형 채널: 10~30% (가끔 의견 표명)
- 미디어: 0~10% (거의 예측 없음)

#### P2: 행동 키워드 강도 (Action Keyword Intensity)

```python
# prediction_detector.py의 BUY/SELL 패턴 활용
action_keywords_per_video = 영상별 (buy_pattern_count + sell_pattern_count) 평균
# 0~5: 낮음, 5~15: 중간, 15+: 높음 → 0~100으로 정규화
```

특히 **직접적 행동 유도** 키워드에 가중치:
- 2배: "사세요", "팔으세요", "들어가세요", "빠지세요", "손절하세요"
- 1.5배: "매수", "매도", "목표가", "진입", "탈출"
- 1배: 나머지 패턴

#### P3: 종목 집중도 (Asset Concentration)

```python
# 특정 종목을 반복적으로 언급 → 리딩방 특성
top5_ratio = (상위 5개 종목 언급 수) / (전체 종목 언급 수) × 100
```

- 예측형: 50~80% (특정 종목 집중 추천)
- 해설형: 20~40% (다양한 종목 언급)

#### P4: 목표가 빈도 (Price Target Frequency)

```python
# 신규 패턴 매칭 필요
price_patterns = [
    r'목표가\s*\d',
    r'\d+원\s*(돌파|이탈|지지|저항)',
    r'\d+\s*(달러|불)\s*(돌파|이탈|도달)',
    r'(TP|타겟|목표)\s*:?\s*\d',
    r'\d+K\s*(돌파|이탈)',  # 코인
]
price_target_ratio = (목표가 패턴 발견 영상 수) / (전체 영상 수) × 100
```

#### P5: 감성 편향도 (Sentiment Bias)

```python
# 중립이 아닌 강한 의견 → 예측형 특성
sentiment_bias = abs(positive_pct - negative_pct) 평균
# 해설형: 30~50% (비교적 균형), 예측형: 60~90% (편향적)
```

## 2. 예측형 채널 발굴 방법

### 2.1 기존 74개 채널 재분류

크롤러가 이미 수집한 데이터로 PIS를 계산하여 자동 분류:

```python
# crawler/channel_classifier.py (신규)
def classify_existing_channels():
    """기존 74개 채널의 최근 30일 데이터로 PIS 산출"""
    for channel in get_all_channels():
        pis = calculate_pis(channel.id, days=30)
        channel_type = determine_type(pis, channel)
        update_channel_classification(channel.id, channel_type, pis)
```

### 2.3 발굴 파이프라인

```
1. 키워드 검색으로 후보 채널 수집 (50~100개)
2. 각 후보의 최근 영상 10개 자막 수집
3. PIS 간이 계산 (P1 + P2만으로 빠른 필터링)
4. PIS >= 40인 채널 → 전체 PIS 계산
5. PIS >= 60인 채널 → predictor로 등록
6. 주 1회 재실행하여 신규 채널 계속 발굴
```
